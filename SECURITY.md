# Security Policy

## 支持范围

安全修复优先应用于 `main` 分支的最新版本。旧提交不承诺持续维护。

## 报告安全问题

请不要通过公开 Issue 披露可利用的漏洞、密钥或个人信息。优先使用仓库 Security 页面提供的私密漏洞报告入口；如果该入口不可用，请通过维护者 GitHub 主页中公开的联系方式联系，并只发送复现所需的最少信息。

报告建议包含：

- 受影响的页面、版本或提交
- 可复现步骤和影响范围
- 已知的缓解方式
- 是否已经公开披露

请勿在未获得授权的情况下访问他人数据、执行破坏性测试或扩大漏洞影响。

## 使用者责任

- 不要在前端代码或 `VITE_` 环境变量中保存秘密。
- 部署平台和 GitHub 凭据遵循最小权限原则。
- 动态 API 应实施鉴权、输入校验、限流和日志脱敏。
- 定期处理依赖更新和安全告警。
